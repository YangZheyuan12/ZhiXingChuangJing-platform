# 知行创境教育创作平台 — 功能质量分析报告

> **分析日期**：2025-04-23  
> **分析范围**：功能实现质量与产品可用性评估（不含代码质量审查）  
> **项目版本**：当前主分支最新代码

---

## 一、项目概况

### 1.1 产品定位

面向中小学的任务驱动式教育创作平台，核心场景为：教师发布创作任务 → 学生创建数字展厅 → 社区互评展示 → 博物馆资源接入 → AI 数字人讲解。

### 1.2 技术架构

| 层级 | 技术栈 |
|------|--------|
| 后端 | Java 17 / Spring Boot / Spring Security / Spring Data JPA / MySQL 8 / Redis 7 |
| 前端 | Vue 3 / TypeScript / Vite / Tailwind CSS / Pinia / Vue Router / Fabric.js |
| 存储 | MinIO 对象存储 |
| 部署 | Docker Compose（MySQL + Redis + MinIO + Nginx） |

### 1.3 功能规模

| 指标 | 数量 |
|------|------|
| 后端 Controller | 13 |
| 后端 Service | 13 |
| 数据库表 | 25 |
| 前端页面视图 | 24 |
| 前端 API 模块 | 12 |
| 前端 Store | 4 |

## 二、功能模块逐项评估

### 2.1 认证与安全 — ⭐⭐⭐⭐（完整可用）

**已实现功能**：
- JWT 双 Token 机制（access token + refresh token），支持过期自动提示
- BCrypt 密码哈希存储，登录时自动升级旧格式密码
- SVG 图形验证码 + Redis 存储（5 分钟有效期），一次验证后即删除防重放
- 前端路由守卫：未登录重定向、已登录跳过登录页、管理员路由隔离
- Spring Security 配置：无状态 Session、CORS、自定义认证入口和拒绝处理器

**权限控制**：
- 三角色模型：admin / teacher / student
- 后端每个业务接口均有角色校验（如仅教师可发布任务、仅班级成员可访问班级）
- 前端路由 meta 标记 `public` / `adminOnly`，守卫层统一拦截

**不足**：
- 注册未实现邮箱/手机号验证（数据库有字段但未使用）
- Token 黑名单/强制下线机制缺失
- 无登录失败次数限制

---

### 2.2 班级管理 — ⭐⭐⭐（基础可用，缺管理入口）

**已实现功能**：
- 查看我的班级列表
- 查看班级详情、成员列表
- 查看班级公告（支持置顶标记）
- 教师/管理员发布公告，含权限校验（非该班教师不可发布）
- 班级访问控制：非班级成员不可访问，管理员不受限制

**不足**：
- 缺少创建/编辑/删除班级的功能
- 缺少邀请/移除学生的功能
- 缺少班级分组管理
- 缺少公告编辑/删除

---

### 2.3 任务教学 — ⭐⭐⭐⭐（核心流程闭环）

**已实现功能**：
- 教师创建任务：标题、描述、截止时间、背景资料（JSON 存储）
- 任务投放到指定班级
- 学生查看任务列表（分页）和详情
- 学生提交作品（关联展厅）
- 教师查看提交列表与进度统计
- 教师评分与点评（`SubmissionReviewEntity`）
- 优秀展厅推选

**不足**：
- 任务状态流转（draft → published → closed → archived）在后端枚举存在，但缺少关闭/归档的 API
- 任务无法编辑已发布的内容
- 缺少批量操作（批量评分、批量导出）
- 缺少成绩汇总导出

---

### 2.4 展厅创作（编辑器）— ⭐（产品核心，但仅为最小 PoC）

**已实现功能**：
- 基于 Fabric.js 的画布编辑器
- 插入文本框（Textbox）
- 插入矩形面板（Rect）
- 插入博物馆资源标题（以纯文本 Textbox 形式）
- 选中元素后可编辑 X/Y/宽/高属性
- 版本保存（手动存草稿）与版本恢复（加载最新版本）
- 发布展厅（保存版本 + 设置 visibility）
- 画布自适应容器尺寸（ResizeObserver）
- 左侧双标签面板（组件库 / 博物馆素材）

**严重缺失**：

| 缺失能力 | 影响 |
|----------|------|
| 图片/视频/音频拖入画布 | 展厅无法包含任何多媒体内容 |
| 图层管理面板 | 元素多时无法管理堆叠顺序 |
| 撤销/重做（Undo/Redo） | 误操作无法恢复 |
| 自动保存 | 后端枚举 `autosave` 存在但前端未实现，断网/崩溃丢失全部工作 |
| 组件模板/预设布局 | 学生需从零开始搭建，门槛高 |
| 对齐/吸附辅助线 | 元素排列全靠手动对齐 |
| 富文本编辑（字体/颜色/对齐） | 文本样式极其有限 |
| 展厅预览/观众浏览模式 | `getViewerData` API 存在但无前端页面 |
| 画布缩放/平移 | 无法处理大画布 |
| 键盘快捷键 | 无 Delete/Ctrl+Z 等 |

**结论**：作为"数字展厅创作平台"的核心差异化功能，当前编辑器仅支持纯文本和矩形两种元素，**不具备实际教学创作能力**。

---

### 2.5 AI 数字人 — ⭐（仅 CRUD 壳，零 AI 能力）

**已实现功能**：
- 为展厅数字人绑定装备（设备名称 + 类型 + 配置 JSON）
- 删除装备
- 权限校验（仅展厅所有者或管理员可操作）

**完全缺失**：

| 缺失能力 | 说明 |
|----------|------|
| TTS 文字转语音 | 无任何语音合成集成 |
| LLM/对话 AI | 无 ChatGPT/文心等 AI 对话接入 |
| 2D/3D 形象渲染 | 无前端渲染组件（`components/digital-human/` 目录为空） |
| 故事脚本播放引擎 | 数据库有 `story_timeline` JSON 字段，但无解释器 |
| 数字人管理页面 | 前端无独立管理界面 |

**结论**：整个"AI 数字人"模块仅是一个关联表的 CRUD，与"AI"无任何实质关系。

---

### 2.6 博物馆资源 — ⭐⭐（查询可用，同步接口为假数据）

**已实现功能**：
- 分页查询博物馆资源列表（支持分类、关键词筛选）
- 查看资源详情（元数据 JSON 解析展示）
- 同步接口存在但实际写入硬编码假数据：

```java
// MuseumServiceImpl.java — 同步方法实际行为
for (int index = 1; index <= 3; index++) {
    metadata.put("keyword", keyword);
    museumRepository.upsertResource(
        providerId,
        "%s-%s-%d".formatted(request.providerCode(), category, index),
        "image", category,
        "%s资源样本 %d".formatted(keyword, index),
        "知行合作馆藏",
        "https://example.com/museum/%s/%d.jpg".formatted(request.providerCode(), index),
        ...
    );
}
```

**缺失**：
- 无真实外部博物馆 API 对接
- 数据库设计了 `cache_status` / `cache_expire_at` 字段但未启用
- 无定时刷新任务（`job/` 目录为空）

---

### 2.7 社区互动 — ⭐⭐⭐⭐（功能丰富，有体验缺陷）

**已实现功能**：
- 社区展厅列表（分页、排序）
- 展厅详情页（统计指标、标签、创作者信息）
- 点赞/取消点赞（乐观更新 + 失败回滚）
- 收藏/取消收藏（同上）
- 分享记录
- 教师/管理员推荐到首页
- 评论系统：
  - 根评论 + 嵌套回复（parentCommentId + rootCommentId）
  - @提及用户
  - 教师点评与学生评论分 Tab 展示
  - 评论自动触发通知
- 长文本折叠/展开

**体验缺陷**：
- **点赞/收藏状态存储在 localStorage**（`CommunityDetailView.vue:606-614`），换浏览器或清缓存即丢失，应由服务端返回
- 评论无分页，全量加载可能有性能问题
- 无评论删除/举报功能

---

### 2.8 素材管理 — ⭐⭐⭐（上传可用，管理有限）

**已实现功能**：
- MinIO 对象存储上传，自动创建 Bucket 并设置公读策略
- 文件类型自动推断（image / video / audio / model / document）
- 头像上传额外校验：仅限图片格式、不超过 5MB
- UUID 文件名防冲突
- 素材列表分页查询（按类型/来源筛选）

**不足**：
- 普通素材上传无文件大小限制（潜在 DoS 风险）
- 无素材删除功能
- 无素材预览功能
- 无素材与展厅的关联管理

---

### 2.9 工作台仪表盘 — ⭐⭐（有 UI，数据不真实）

**已实现功能**：
- 时段问候语（凌晨/上午/下午/晚上）
- 进行中任务卡片列表（前端分页）
- 班级公告折叠面板
- 统计指标卡片
- 快捷入口导航（根据角色动态显示）

**严重问题**：

1. **任务进度条是伪随机数**：
```javascript
function getTaskProgress(task: TaskSummary) {
  const seed = task.id * 17 + task.title.length * 7
  return 38 + (seed % 48)  // 永远在 38%~86% 之间，与实际无关
}
```

2. **API 失败时回退到硬编码 mock 数据**（约 100 行），以真实数据的 UI 样式展示给用户，容易造成误导。

## 三、系统级缺陷

### 3.1 严重问题（影响生产可用性）

| # | 问题 | 详情 |
|---|------|------|
| 1 | **零自动化测试** | 后端仅有一个空壳测试类 `ZhiXingChuangJingApplicationTests.java`，前端无任何测试。任何修改都可能引入不可察觉的回归 |
| 2 | **无 WebSocket 实时通信** | `websocket/` 目录为空。展厅编辑器无法多人协作，通知无法实时推送 — 但数据库已设计了 `exhibition_members` 协作表 |
| 3 | **无 API 速率限制** | 所有接口无 rate limiting，评论/点赞/注册均可被脚本无限请求 |
| 4 | **无内容安全** | 评论内容直接入库展示，无敏感词过滤、无 HTML/XSS 过滤（前端未使用 `v-html`，暂不构成 XSS 漏洞，但缺乏纵深防御） |
| 5 | **无定时任务引擎** | `job/` 目录为空，博物馆缓存刷新、统计数据校正、过期任务关闭等异步逻辑均未实现 |
| 6 | **交互状态客户端存储** | 社区点赞/收藏状态存 localStorage 而非来自服务端，跨设备不同步 |

### 3.2 中度问题（影响用户体验与可扩展性）

| # | 问题 | 详情 |
|---|------|------|
| 1 | Redis 利用率极低 | 仅验证码使用 Redis，无 API 缓存、无分布式锁、无热点数据缓存 |
| 2 | 评论无分页 | 全量加载评论列表，数据量增长后性能下降 |
| 3 | 无管理后台 | admin 路由仅有一个 Dashboard 视图，缺少用户管理、内容审核、系统配置 |
| 4 | 无邮箱/手机验证 | 数据库有字段但注册时不验证 |
| 5 | 无数据导出 | 教师无法导出成绩单、提交汇总 |
| 6 | 展厅无独立浏览页 | `getViewerData` API 存在但无对应前端页面 |
| 7 | 多处空目录 | `job/`、`mapper/`、`router/`、`websocket/`、`composables/`、`types/`、`digital-human/`、`exhibition/` 均为空 |

## 五、总体结论

### 5.1 能否经受考验、广泛使用？

**答案：目前不能。**

系统的 **"外围功能"**（认证、班级、任务、社区互动、通知）做得相对完整，API 设计规范，权限控制到位，数据库设计专业。但其 **三个产品差异化核心**：

1. **展厅编辑器** — 仅支持纯文本和矩形，无图片/视频/模板
2. **AI 数字人** — 仅有装备绑定 CRUD，无任何 AI 能力
3. **博物馆资源接入** — 同步接口写入硬编码假数据

**全部处于半成品或空壳状态**。

### 5.2 适用场景判断

| 场景 | 是否适用 |
|------|----------|
| 计算机设计大赛演示答辩 | ✅ 架构完整、UI 精良，可完成演示流程 |
| 真实课堂教学使用 | ❌ 编辑器无法创建有意义的展厅内容 |
| 规模化部署（>100 用户） | ❌ 无速率限制、无缓存、无内容审核、零测试 |
| 产品原型验证 | ✅ 可作为 MVP 验证教学场景需求 |

### 5.3 要达到"可广泛使用"需要的最小改进

1. **展厅编辑器**：支持图片/视频上传到画布、图层管理、撤销重做、自动保存、预设模板
2. **AI 数字人**：至少接入一个 TTS API + 简单的台词播放前端
3. **博物馆资源**：对接至少一个真实公开数据源（如故宫开放平台）
4. **系统基建**：API 速率限制、评论分页、交互状态服务端化、基础自动化测试
